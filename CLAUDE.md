## Project Overview

Next.js 16 (App Router, Turbopack) theme park data platform. Supports 6 locales (en, de, fr, it, nl, es). Live wait times, crowd calendars, glossary, park/attraction detail pages.

## Commands

```bash
npm run dev          # Start dev server (Turbopack)
npm run build        # Full build (includes prebuild scripts)
npm run lint         # ESLint
npm run lint:fix     # ESLint with auto-fix
npm run format       # Prettier write
npm run format:check # Prettier check
npm run validate:translations  # Check translation completeness
npx tsc --noEmit     # Type-check only
```

## Glossary System

### Data flow

```
lib/glossary/data.ts          // static term metadata (id, category, slugs per locale)
content/glossary/{locale}.ts  // translations (name, shortDefinition, definition)
lib/glossary/translations.ts  // getGlossaryTerms(locale) â€” React cache()
```

### Glossary Term Inject (automatic linking)

Wraps body text to auto-highlight and link glossary terms (first occurrence per term, longest-match first, case-insensitive).

```tsx
// 1. Wrap the page (server component) â€” fetches terms once, provides context
import { GlossaryInjectLoader } from '@/components/glossary/glossary-inject-loader';

<GlossaryInjectLoader locale={locale}>
  {children}
</GlossaryInjectLoader>

// 2. Use anywhere inside (client or server) â€” no props needed
import { GlossaryInject } from '@/components/glossary/glossary-inject';

<GlossaryInject>{someTextString}</GlossaryInject>
```

Currently wired into: attraction page FAQ answers (`components/faq/attraction-faq-section.tsx`).

### Adding a new glossary term

1. Add `{ id, category, slugs }` entry to `lib/glossary/data.ts`
2. Add translation object `{ id, name, shortDefinition, definition }` to each `content/glossary/{locale}.ts`
3. Update `GLOSSARY_CONTENT_DATE` in `components/seo/glossary-structured-data.tsx`
